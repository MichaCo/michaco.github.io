WithBackPlateTBackPlate
Configures the back plate for the cache manager.

This is an optional feature. If specified, see the documentation for the TBackPlate. The name might be used to reference another configuration item.

If a back plate is defined, at least one cache handle must be marked as back plate source. The cache manager then will try to synchronize multiple instances of the same configuration.

WithMaxRetries
Sets the maximum number of retries per action.

Default is MaxValue.

Not every cache handle implements this, usually only distributed caches will use it.

WithRetryTimeout
Sets the timeout between each retry of an action in milliseconds.

Default is 10.

Not every cache handle implements this, usually only distributed caches will use it.

WithUpdateMode
Sets the update mode of the cache.

If nothing is set, the default will be CacheUpdateMode.None.

WithRedisBackPlate
Configures the back plate for the cache manager.

The redisConfigurationId is used to define the redis configuration, the back plate should use to connect to the redis server.

If a back plate is defined, at least one cache handle must be marked as back plate source. The cache manager then will try to synchronize multiple instances of the same configuration.
